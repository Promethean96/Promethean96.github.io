HMTL Elements
Element Description
Tip:
<!--...--> Used to insert comments in the source code. Comments are not displayed in the browsers.
<a=> Defines a hyperlink, which is used to link from one page to another.
You must include a reference attribute as well.
<abbr> Defines an abbreviation or acronym.
Use the global <title> attribute to show the description for the abbreviation or acronym when you mouse over the element.
<address> Defnies the contact information for the author/owner of a document or article.
The text in this element usually renders in italic and browsers will always add a line break before and after.
It is recommended to use <br> on each line of contact
<area> Defines an area inside an image map.
This element is always nested inside a <map> tag
The usemap attribute in <img> is associated with the <map> element's name attribute, and creates a relationship between the image and the map.
<article> Specifies independent, self-contained content. An article should make sense on its own.
This element does not render anything special but can be styled using CSS.
<aside> Defines some content aside from the content it is placed in.
This element is often placed as a sidebar in a document. It does not render anything special but can be styled with CSS.
<audio> Used to embed sound content in a document, such as music or audio streams. Must containt a source attribute.
HTML supports MP3, WAV, and OCG. Safari does not support OCG.


HMTL Attributes
Attriblue Description
Tip:
href= Specifies the URL of the page the reference goes to.
src= Specifiec the source/path to the image being displayed.
Absolute URL link to external sources and should be avoided. Relative URL link within your site and should be used.
width= or height= Specify which Width or Height the image should be.
It is recommended to use at least one of these for every <img>
alt= Specifies an alternate text for an image
If the image cannor be displayed, due to an error or slow connection, this will be displayed. This will also be used for screen reader applications.
style= Attribute to apply styles to an element, such as color, font, size, and more.
Please also refer to styles table.
lang= Defines the language of the web page.
You should always include a language in your web code.
title= Defines some extra information about an element.
The value of the title attribute will be displayed as a tooltip when you mouse over the element.
autoplay Specifies that the audio will start playing as soon as it is ready.
controls Specifies that the audio controls should be displayed. Such as a play/pause button.
loop Specifies that the audio will start over again, every time it is finished.
muted Specifies that the audio output should be muted.
preload= Specifies if and how the author thinks the audio should be loaded when the page loads. Value can be auto, metadata, or none.


HMTL Table Reference
Selector Meaning Description
Tip:
<table> Table Defines an HTML table
</table>
<thead> Table Head Defines header content in an HTML table.
The <thead> elements must have one or more <tr> tags inside.
</thead>
<caption> Table Caption Defines a table caption.
By default, a table caption will be center-aligned above a table.
</caption>
<colgroup> Column Group Defines a group of one or more columns in a table for formatting.
To define different properties to a column within a <colgroup>, use the <col> tag within the <colgroup> tag
</colgroup>
<tbody> Table Body Defnies the body content in an HTML table.
The <tbody> elements must have one or more <tr> tags inside.
</tbody>
<tfoot> Table Foot Defines a footer content in an HTML table.
The <tfoot> elements must have one or more <tr> tags inside.
</tfoot>
<th> Table Header The table header container, bolded and center
The <th> elements are bold and centered by default.
</th>
<tr> Table Row The table row container
The <tr> elements must have one or more <th> or <td> tags inside.
</tr>
<td> Table Data The table data container
The <td> elements are regular and left-aligned by default
</td>
